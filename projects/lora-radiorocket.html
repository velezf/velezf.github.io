Overview

A ground-up rewrite of N3VEM’s RadioRocketV2 for the Adafruit Feather M0 + RFM95 platform. The rocket sled reads barometric altitude and high-G acceleration, detects launch and apogee autonomously, and streams 1 Hz telemetry packets to a matching Feather M0 ground station over 434 MHz LoRa.

The rocket is a PiercerMini — a mid-power airframe with the electronics bay housing the sled.

Hardware

TX Sled (rocket)

Component Purpose
Adafruit Feather M0 + RFM95 (434 MHz) Microcontroller + LoRa radio
Adafruit BMP390 Barometric pressure / altitude
Adafruit ADXL375 High-G accelerometer (±200 g)
LiPo battery Power

RX Ground Station

The ground station is a LePotato SBC built into a custom blue acrylic enclosure. It serves double duty: receiving LoRa telemetry from the rocket and acting as a portable APRS digipeater — because most mid-power launch sites are outside range of any APRS I-Gate or digipeater, this station brings the infrastructure to the field.

Component Purpose
Libre Computer LePotato SBC Runs Linux, Direwolf, Node-RED
RTL-SDR V4 dongle APRS receive / digipeater (144.39 MHz)
Adafruit Feather M0 + RFM95 (434 MHz) LoRa telemetry receiver
12V → 5V DC-DC converter Field power from vehicle or battery
Custom blue acrylic enclosure Houses all hardware

Software stack: Direwolf (software TNC + digipeater), Xastir (APRS mapping), Node-RED (telemetry dashboard), VNC (headless remote access).

When the rocket eventually carries an APRS tracker (planned: QRPLabs LightAPRS 2.0), this station will digipeat its packets to the APRS network from the field.

Firmware

Three algorithms do the interesting work:

1. Ground pressure calibration

At boot, the sled takes 50 pressure readings over ~2.5 seconds and averages them to establish a stable ground-level reference. Everything else — altitude, apogee — is computed relative to this baseline.

// Calibrate ground pressure
float sum = 0;
for (int i = 0; i < 50; i++) {
  bmp.performReading();
  sum += bmp.pressure / 100.0;
  delay(50);
}
groundPressure = sum / 50.0;

A simple average works well on a calm pad. A future improvement would be a trimmed mean — discarding the top and bottom few samples — to handle sensor noise spikes at power-on.

2. Launch detection

Rather than checking a single axis, launch is detected from the vector magnitude across all three accelerometer axes. This makes detection robust regardless of how the sled is oriented in the airframe.

float g = sqrt(event.acceleration.x * event.acceleration.x +
               event.acceleration.y * event.acceleration.y +
               event.acceleration.z * event.acceleration.z) / 9.80665;

if (!inFlight && g > launchThresholdG) {
  inFlight = true;
  launchTime = now;
}

3. Apogee detection

Apogee is detected purely from barometric altitude — no additional sensors needed. The firmware tracks the running maximum altitude and flags descent the moment altitude stops climbing.

if (altitudeFt > maxAltitudeFt) {
  maxAltitudeFt = altitudeFt;
  descending = false;  // still ascending
} else {
  descending = true;   // apogee passed
}
if (g > peakG) peakG = g;  // update peak G

Each 1 Hz telemetry packet bundles all of this into a compact string — current altitude, max altitude, live G, peak G, temperature, battery voltage, and flight state — and waits up to 300 ms for an ACK from the ground station before continuing.
